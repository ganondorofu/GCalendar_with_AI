# GoogleカレンダーwithAI詳細設計書

## システム概要
本システムは、生成AI（Gemini API）を活用して自然言語でGoogleカレンダーを操作できるアプリケーションである。MVCアーキテクチャを採用し、WebサイトおよびDiscord/Line Botから利用可能。ユーザー認証はGoogle OAuth2.0を使用。

## アーキテクチャ設計
- **全体構造**: Spring BootベースのMVCモデル。
  - **Model**: データ処理（Google Calendar API、Gemini API、MySQL）。
  - **View**: Thymeleafテンプレート（Web UI）。
  - **Controller**: リクエスト処理とレスポンス生成。
- **外部連携**:
  - Google Calendar API: 予定の追加/確認。
  - Gemini API: 自然言語解析。
  - Discord/Line Bot: Webhook経由の連携。
- **データフロー**: ユーザー入力 → Controller → Service（AI解析） → Model（API呼び出し） → View/Botレスポンス。

## 機能詳細設計

### 予定追加機能
- **処理フロー**:
  1. ユーザー入力（自然言語）を受け取る。
  2. Gemini APIで日時・内容を解析。
  3. Google Calendar APIで予定を追加。
  4. 成功メッセージを返す。
- **クラス設計**:
  - `CalendarService`: AI解析とAPI呼び出し。
  - `MainController`: エンドポイント `/add-event`。

### 予定確認機能
- **処理フロー**:
  1. ユーザー入力を受け取る。
  2. Gemini APIでクエリを解析。
  3. Google Calendar APIで予定を取得。
  4. 予定リストを返す。
- **クラス設計**:
  - `CalendarService`: 同様。
  - `MainController`: エンドポイント `/get-events`。
- **シーケンス図**: (Draw.ioで作成予定)

### ユーザー認証機能
- **処理フロー**:
  1. Google OAuth2.0で認証。
  2. トークンを取得し、セッションに保存。
- **クラス設計**:
  - `AuthController`: 認証処理。
  - セキュリティ: Spring Security使用。

## データベース設計
- **テーブル**:
  - `users`: id, google_id, access_token (暗号化)。
  - `events`: id, user_id, title, start_time, end_time。
- **ER図**: (Draw.ioで作成予定)

## API設計
- **内部API**:
  - POST /api/add-event: 予定追加。
  - GET /api/get-events: 予定確認。
- **Bot連携**: Webhook URLでPOSTリクエストを受け、レスポンスをBotに返す。

## UI設計
- **Web UI**: Thymeleafでシンプルなフォーム（入力フィールド、ボタン）。
- **Bot UI**: テキストベースのインタラクション。

## セキュリティ設計
- **認証**: OAuth2.0。
- **データ保護**: トークンはAES暗号化、DBはハッシュ化。
- **HTTPS**: Koyebデプロイ時に有効化。

## テスト設計
- **単体テスト**: JUnitで各Serviceクラス。
- **統合テスト**: APIエンドポイントのテスト。
- **テストケース例**: 予定追加の正常系/異常系。
